# Follow-up Roadmap (Research Iteration 2)

This roadmap consolidates the second-pass research into concrete, implementation-ready work items.

## Priority 1 — Consumer validation path

1. Add `apps/web` consumer scaffold to validate package integration end-to-end.
2. Demonstrate light/dark token consumption with real routes/components.

## Priority 2 — Primitive expansion

1. Add Input + Field primitives with accessibility-first contracts.
2. Add Dialog + Menu primitives with consistent slot/variant rules.

## Priority 3 — Token pipeline maturity

1. Add token snapshots/versioning docs to track visual contract changes.
2. Add generated artifact checks to ensure token CSS stays deterministic.

## Priority 4 — Documentation depth

1. Keep source-by-source research files under `docs/research/*` updated.
2. Add a component contract catalog (props, variants, accessibility semantics).

## Priority 5 — Quality and release hardening

1. Add visual regression checks for primitives.
2. Add accessibility checks in CI for new components.
