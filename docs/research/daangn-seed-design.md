# Research: daangn/seed-design

Source: https://github.com/daangn/seed-design

## Observed signals

- Large monorepo with `packages`, `docs`, `ecosystem`, and integration-focused packages.
- README separates definitions, base CSS, React headless/react packages, and integrations.
- Strong signal for long-lived design-system package layering.

## Applied takeaways

- Keep architecture layered (definitions/tokens, UI, integrations/config).
- Avoid mixing foundational contracts with framework adapters.
- Document boundaries early to reduce future coupling.
