# Research: multica/apps/web

Source: https://github.com/multica-ai/multica/tree/main/apps/web

## Observed signals

- Feature-oriented structure (`app`, `components`, `features`, `shared`, `lib`, `hooks`).
- Next.js app with typed config and dedicated test setup.
- Strong separation between UI primitives and product feature modules.

## Applied takeaways

- Keep design-system primitives in dedicated packages and avoid feature coupling.
- Prefer semantic token-driven styling and reusable class composition.
- Preserve a clean, modern, structured visual language.
